# Forsaken Realms AI - Local Obsidian Knowledge Assistant

Forsaken Realms AI is a local-first AI assistant designed to turn an Obsidian vault into an interactive knowledge system. It combines Python, PySide6, Ollama, and vector search to create an AI companion that can read, understand, and reason over interconnected notes.

The project was originally built for the **Forsaken Realms** worldbuilding vault but is designed to be adaptable to any Obsidian knowledge base, including writing projects, learning repositories, personal wikis, research collections, and documentation systems.

## Features

* Local-first architecture with no cloud dependency
* Integration with Ollama and local language models (currently Qwen3:8B)
* Obsidian vault indexing and semantic search
* Automatic note ingestion and vector database generation
* Real-time vault monitoring and automatic index updates
* Knowledge graph traversal through Obsidian wiki links
* Context-aware question answering based on vault content
* Writing assistance and live suggestions while drafting notes
* Portable setup wizard allowing users to choose their own vault and data locations
* Packaged desktop application built with PySide6 and PyInstaller

## Project Goals

Forsaken Realms AI aims to evolve beyond a simple chatbot into a persistent knowledge companion that grows alongside its user's vault. As notes, relationships, and information expand over time, the assistant becomes increasingly capable of:

* Exploring relationships between notes
* Answering questions using contextual knowledge
* Assisting with writing and worldbuilding
* Acting as a personal learning and research companion
* Serving as an intelligent interface to a user's entire knowledge system

## Technology Stack

* Python 3
* PySide6
* Ollama
* Qwen3:8B
* Sentence Transformers
* FAISS Vector Search
* Obsidian Markdown Vaults
* PyInstaller
* Inno Setup

## Current Status

This project is actively under development and serves as both a learning project and an experimental local AI platform. The long-term vision is to create a fully featured knowledge assistant that remains entirely under the user's control and can scale from small note collections to massive interconnected knowledge bases.
